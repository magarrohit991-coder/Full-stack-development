01 — Overview

WHAT IS FULL STACK DEVELOPMENT?

Full stack development means working across all layers of a web application — from what users see in their browser (front-end) to the server, database, and infrastructure behind the scenes (back-end). A full stack developer can design, build, and deploy a complete web application independently.

"A full stack developer understands both sides of the web — the experience users have, and the logic that makes it possible."

— Core principle of modern web engineering
01

Front-End

Everything the user sees — buttons, forms, layouts, animations. Built with HTML, CSS, JavaScript and frameworks like React or Vue. The goal is a fast, accessible, and beautiful interface.

HTML5CSS3JavaScriptReact
02

Back-End

The server-side logic — handles requests, processes data, manages authentication, and talks to the database. Built with Node.js, Python, PHP or Java and frameworks like Express or Django.

Node.jsExpressPythonREST API
03

Database

Stores all application data persistently. Relational databases (MySQL, PostgreSQL) use tables and SQL queries. NoSQL databases (MongoDB) use flexible document-based storage.

MySQLMongoDBPostgreSQL
04

APIs

APIs allow front-end and back-end to communicate. REST APIs use HTTP methods (GET, POST, PUT, DELETE) to transfer JSON data. GraphQL lets clients request exactly what they need.

RESTGraphQLJSONHTTP
05

DevOps

Once built, apps must be deployed and maintained. Git tracks code changes, Docker containerises the app, and cloud platforms like AWS or Netlify host it. CI/CD automates testing and deployment.

GitDockerAWSCI/CD
06

Security

Developers must guard against SQL injection, XSS, and CSRF attacks. HTTPS encrypts data in transit, JWT handles authentication, and input validation prevents malicious data from entering the system.

HTTPSJWTValidation

Real-World Example — How a Login Works

Step 1 — Front-End: The user fills in an email and password in an HTML form and clicks "Login". JavaScript captures the form data and sends it as a POST request via fetch('/api/login').

Step 2 — Back-End: The Node.js/Express server receives the request, validates the input, and queries the database: SELECT * FROM users WHERE email = ?.

Step 3 — Database: MySQL returns the matching user record. The server compares the hashed password using bcrypt.

Step 4 — Response: If the password matches, the server creates a JWT token and sends it back. The front-end stores the token and redirects the user to their dashboard — all without a full page reload.

02 — Markup Language

HTML

HyperText Markup Language is the foundation of every webpage. It provides structure and meaning through elements and tags. HTML tells the browser what type of content is on the page — a heading, paragraph, image, button, or form. Without HTML, there is no webpage.

Tags, Elements & Attributes

HTML uses tags wrapped in angle brackets: <p>. Most tags come in pairs — an opening tag and closing tag. The content between them is the element. For example: <h1>Hello</h1> creates a top-level heading.

Attributes add extra information inside the opening tag. <a href="url">Link</a> uses href to define the destination. <img src="photo.jpg" alt="Me"> embeds an image with a text description for screen readers.

Every HTML page has a standard skeleton: <!DOCTYPE html> declares the version, <head> holds metadata, and <body> holds everything the user sees.

<h1> Page Title </h1>
<p> Paragraph text here. </p>
<a href="#">Click me</a>
<img src="photo.jpg" alt="Me">
Example — Semantic Page Structure

Semantic HTML5 Elements

Semantic HTML uses elements that carry meaning — not just visual styling. Instead of wrapping everything in generic <div> tags, semantic HTML uses purpose-built elements that describe the content's role.

<header> — the top of the page or section.
<nav> — navigation links.
<main> — the primary content area.
<article> — a self-contained piece of content.
<aside> — supplementary content.
<footer> — the bottom of the page.

Benefits: better SEO (search engines understand content), better accessibility (screen readers navigate by landmarks), and better maintainability (developers understand structure at a glance).

Example — HTML Form with Validation

HTML5 Forms — Collecting User Input

Forms allow users to send data to a server. HTML5 introduced built-in validation: type="email" validates email format, required prevents empty submission, minlength enforces minimum length, and placeholder shows hint text — all without any JavaScript.

Every input must have a matching <label> linked via the for attribute. This improves accessibility — clicking the label focuses the input — and helps screen readers describe the field to visually impaired users.

The method attribute on <form> defines how data is sent: GET appends data to the URL (good for searches), POST sends it in the request body (good for sensitive data like passwords).

03 — Styling & Layout

CSS

Cascading Style Sheets (CSS) controls the visual presentation of HTML. While HTML creates structure, CSS transforms it into a beautifully designed, responsive interface. CSS controls colours, typography, spacing, layout, animations, and how content adapts across screen sizes.

Selectors, Properties & the Cascade

A CSS rule has two parts: a selector (which element to target) and a declaration block (what styles to apply). For example: h1 { color: white; font-size: 48px; } makes all h1s white and large.

The "Cascading" means when multiple rules target the same element, the browser resolves conflicts using specificity (more specific selectors win), source order (later rules override earlier ones), and importance (!important overrides all).

CSS is applied three ways: Inline (style="" on an element), Internal (a <style> block in <head>), or External (a separate .css file linked via <link>). External is the professional standard.

h1 { color: white; }
.card { background: #111; }
#logo { font-size: 24px; }
a:hover { color: #F5C518; }
Example — Flexbox Layout

Flexbox — One-Dimensional Layout

Flexbox arranges items in a single row or column. Set display: flex on a container and its children become flex items that flow side by side automatically.

Key properties: justify-content aligns items horizontally (center, space-between, flex-end), align-items aligns vertically, gap sets spacing, and flex-wrap: wrap allows items to wrap to the next line on small screens.

Example use cases: navigation bars, button groups, card rows, centering content in a container — any layout where items sit side by side.

display:flex; gap:8px — hover each box to scale

justify-content: center · align-items: center
Example — CSS Grid Layout

CSS Grid — Two-Dimensional Layout

CSS Grid controls rows and columns simultaneously — perfect for page layouts, dashboards, and galleries. Use display: grid to activate it.

grid-template-columns: repeat(3, 1fr) creates 3 equal columns. Items can span multiple columns using grid-column: span 2. The gap property sets spacing between all cells.

Rule of thumb: Use Grid for the overall page structure (header, sidebar, main, footer). Use Flexbox for aligning items within individual components (nav links, button rows).

grid-template-columns: repeat(3, 1fr); gap: 4px

Example — CSS Variables & Transitions

CSS Custom Properties & Transitions

CSS Variables (Custom Properties) store reusable values. Defined once in :root { --accent: #F5C518; } and used anywhere as color: var(--accent). Change the value in one place and every element that uses it updates — essential for consistent theming.

Transitions animate property changes smoothly. transition: all 0.3s ease means any change (on hover, focus, or class toggle) will animate over 0.3 seconds instead of snapping instantly — creating a polished, professional feel.

Example: The three boxes on the right all use transitions. Hover each one to see a different effect — border-radius morph, scale + glow, and colour fill.

hover each box — transition: all 0.3s ease

04 — CSS Framework

BOOTSTRAP FRAMEWORK

Bootstrap is the world's most popular open-source CSS framework, originally created by Twitter. It provides a comprehensive library of pre-built components, a 12-column responsive grid, and utility classes — allowing developers to build professional, mobile-ready websites rapidly without writing CSS from scratch.

Why Use Bootstrap?

Writing CSS for every project from scratch is extremely time-consuming. Bootstrap solves this by providing a consistent, well-tested design system. You simply apply pre-defined class names to HTML elements and Bootstrap handles the styling.

Bootstrap is mobile-first — styles target small screens by default and scale up for larger screens using breakpoints (sm, md, lg, xl, xxl). This aligns with modern best practices: over 60% of web traffic comes from mobile.

To use Bootstrap, add one line to your HTML <head>: <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3/dist/css/bootstrap.min.css" rel="stylesheet">. No setup or build tools needed — start using classes immediately.

GRID

12-Column Grid System

Bootstrap divides the page into 12 equal columns. Assign spans using classes like col-4 (4 of 12 = one third). Breakpoint suffixes like col-md-6 make columns responsive — half width on medium screens, full width on mobile automatically.

col-4col-md-6col-sm-12row
COMP

Pre-Built Components

Bootstrap includes dozens of ready-to-use components: navbars, buttons, cards, modals, dropdowns, alerts, badges, carousels, accordions, and more. All styled consistently and accessible. No custom CSS needed — just add the class name.

btncardmodalnavbarbadge
UTIL

Utility Classes

Bootstrap 5 includes utility classes for rapid styling without any custom CSS. mt-3 adds top margin, d-flex enables flexbox, text-center centres text, fw-bold makes text bold, bg-dark sets background.

05 — Programming & Interactivity

JAVASCRIPT

JavaScript (JS) is the programming language of the web. While HTML builds structure and CSS handles appearance, JavaScript adds dynamic behaviour, logic, and interactivity. It runs directly in the browser and is the only language natively understood by all browsers worldwide.

Core Concepts — Variables, Functions & Events

Variables store data. Modern JS uses let (can change) and const (cannot be reassigned). Functions are reusable blocks of code — declared with function name() {} or as arrow functions: const greet = () => {}.

Events are things that happen in the browser — clicks, keypresses, form submissions. You "listen" for them using element.addEventListener('click', handler). When the event fires, your handler function runs — this is how all interactive web pages work.

The DOM (Document Object Model) is a live tree of all HTML elements. JavaScript can find any element using document.querySelector(), then change its text, style, classes, or attributes in real time — without reloading the page.

const btn = document.querySelector('#btn');
let count = 0;
btn.addEventListener('click', () => {
count++;
btn.textContent = `Clicked ${count}x`;
});

Async JavaScript — Fetch API & Promises Explained

Modern web apps constantly load data from servers — user profiles, posts, product listings — without reloading the entire page. This is done with asynchronous JavaScript. Instead of blocking the browser while waiting for a server response, async code continues running and processes the result when it arrives.

The Fetch API makes HTTP requests. fetch(url) returns a Promise — a placeholder for a future value. Using async/await makes Promises readable: const res = await fetch(url) pauses execution until the response arrives, then const data = await res.json() parses the JSON body.

Example 4 above fetches a random user from the public randomuser.me API — demonstrating a real network request, JSON parsing, and a DOM update all working together. This is exactly how modern front-ends communicate with back-end REST APIs.
